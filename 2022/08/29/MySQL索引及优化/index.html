MySQL索引及优化

发表于2022-08-29|更新于2022-08-29
|阅读量:

本部分内容从官方文档搬运，仅用于学习、记录。以官方文档为准:MySQL :: MySQL 8.0 Reference Manual :: 8.3.1 How MySQL Uses Indexes

概念

索引用于快速查找据有特定的列值所在的行。如果没有索引，MySQL一定会从第一行开始，遍历全表搜索相关的行，如果表很大，那么这个步骤耗时就越久。如果表中存在索引，那么MySQL就可以快速定位到数据文件中间的查询的位置，而不用去查询所有的数据。

这点很好理解，想象你拿到了一本字典，现在让你不查看目录找到某一个字所在的位置，那就只能从头开始一页页查找。如果这个字还是多音字，为了不避免遗漏，需要从头找到尾。如果给字典加上索引(目录)，那么我们直接在目录中找到这个字所在的页码，然后到对应的页码找到这个字，效率比从头翻到尾快上不少。

分类

主键索引(唯一且非空)

唯一索引（唯一可为空）

普通索引（普通字段索引）

全文索引（一般是char、varchar、text类型）

组合索引

数据结构

MySQL的索引用到的数据结构多种多样，可以总结为下面几点：

1、大部分的索引(主键索引、唯一索引、普通索引和全文索引)保存在B+树

扩展：为什么采用B+树？

B+树能显著减少IO次数，提高效率

B+树的查询效率更加稳定，因为数据放在叶子节点

B+树能提高范围查询的效率，因为叶子节点指向下一个叶子节点

B+ Tree Visualization (usfca.edu)

2、空间数据类型的索引采用R树

3、MEMROY搜索引擎支持利用哈希进行索引

4、InnoDB的全文索引采用倒排链表

使用索引的操作

MySQL中使用索引的操作有：

1、快速查到和WHERE语句匹配的行

2、如果有多个索引，那么MySQL通常会选择查找到的行数最少的索引

3、如果表中有多列索引，那么索引可以从任意从最左边的前缀进行查询

4、在联表时从其他表中索引。如果这两列是同样类型和大小，那么MySQL可以更有效地使用索引。

5、查找特定索引列的最大值和最小值。

6、在可用索引的最左前缀上进行排序或分组。

7、在某些情况下，可以在不查阅数据行的情况下检索值。 (覆盖索引)
